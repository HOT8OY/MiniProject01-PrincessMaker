# 프린세스 메이커 게임 프로젝트 개요

---

## 1. 프로젝트 기본 정보

- 프로젝트 명: Princess Maker (프린세스 메이커)
- 개발 언어: Java
- 개발 환경: IntelliJ IDEA, Console 기반 실행
- 프로젝트 설명: 24개월 동안 프린세스를 육성하여 다양한 엔딩을 달성하는 육성 시뮬레이션 게임

---

## 2. 게임 개요

### 게임 장르

- 육성 시뮬레이션 / 텍스트 기반 어드벤처

### 게임 목표

- 프린세스를 24개월(24턴) 동안 육성하여 원하는 엔딩에 도달하기

### 게임 스토리

반역자들에 의해 왕국이 함락되고 왕과 왕비가 폐위되었다. 충직한 신하였던 당신은 아기 공주를 구출해 옆 왕국으로 도망쳤다. 공주의 신분을 숨기고 키워온 지 이제 2년 후면 그녀는 성인이 된다. 성인이 되면 출생의 비밀을 알려줄 것을 결심한 당신은, 공주가 왕국의 공주로서 당당히 설 수 있도록 다양한 능력을 길러주고자 한다.

### 플레이 방식

- 매 턴마다 3가지 활동 중 하나를 선택
- 활동에 따라 능력치가 상승
- 12턴 후 직업을 선택하여 특정 능력치에 보너스 획득
- 최종 능력치에 따라 12가지 엔딩 중 하나로 분기

### 게임 기간

- 총 24턴 (1턴 = 1개월)
- 시작: 2025년 1월
- 종료: 2026년 12월

## 3. 핵심 게임 시스템

### 3-1. 능력치 시스템

| 능력치 | 설명 | 초기값 |
| --- | --- | --- |
| **체력** (Physical) | 육체적 활동과 관련된 능력 | 0 |
| **지능** (Intelligence) | 학습 및 지적 활동과 관련된 능력 | 0 |
| **매력** (Charm) | 사교 및 외모와 관련된 능력 | 0 |

증가 방식:

- 활동 선택 시 랜덤하게 증가 (주 능력치 7~12, 부 능력치 2~4)
- 직업 보너스 적용 시 특정 능력치 50% 추가 증가

---

### 3-2. 활동 시스템

| 활동 | 주 증가 능력치 | 부 증가 능력치 | 설명 |
| --- | --- | --- | --- |
| **운동** | 체력 ↑↑ (7~12) | 지능 ↑ (2~4) | 육체적 단련 활동 |
| **공부** | 지능 ↑↑ (7~12) | 매력 ↑ (2~4) | 학습 및 지적 활동 |
| **사교** | 매력 ↑↑ (7~12) | 체력 ↑ (2~4) | 사람들과의 교류 활동 |

활동 실행 프로세스:

1. 3가지 활동 중 선택 (잘못된 입력 시 재입력 요구)
2. 활동 진행 애니메이션 출력 (점 애니메이션)
3. 결과 출력 (대성공! 능력치 상승)
4. 현재 능력치 표시
5. Enter 입력으로 다음 턴 진행

**입력 검증**:

- 숫자 1, 2, 3만 입력 가능
- 잘못된 입력 시 오류 메시지 및 재입력 요구
- InputMismatchException 처리

---

### 3-3. 직업 시스템

**선택 시점**: 12턴 완료 후

| 직업 | 보너스 능력치 | 보너스 효과 | 특징 |
| --- | --- | --- | --- |
| **군인** (Soldier) | 체력 | +50% | "군인답게 ~을 수행했다!" |
| **학자** (Scholar) | 지능 | +50% | "학자답게 ~을 수행했다." |
| **연예인** (Celebrity) | 매력 | +50% | "연예인답게 ~을 수행했다~" |

**직업 효과**:

- 보너스 능력치와 활동의 주 증가 능력치가 일치할 때 50% 추가 증가
- 예: 군인이 운동을 하면 체력이 10~18 증가 (기본 7~12의 150%)

**직업 선택 연출**:

- 로딩 바 애니메이션 ("취 직 중")
- 직업 선택 완료 메시지
- Enter 키로 계속 진행

## 4. 게임 진행 흐름

```
[시작]
   ↓
[타이틀 화면 - ASCII 아트]
   ├─ Princess Maker 로고
   └─ "PRESS ENTER TO START"
   ↓
[이름 설정]
   ├─ 빈 이름 입력 시 재입력 요구
   └─ 유효한 이름 입력
   ↓
[프롤로그 (스토리 소개)]
   ├─ 환영 메시지
   ├─ 로딩 애니메이션 (8초)
   └─ 스토리 텍스트 (자동 진행, 2초 간격)
   ↓
[1~12턴: 활동 선택 반복]
   ├─ 현재 날짜 표시 (yy년MM월)
   ├─ 활동 선택 (운동/공부/사교)
   │   └─ 잘못된 입력 시 오류 메시지 및 재입력
   ├─ 활동 진행 애니메이션
   ├─ 활동 실행 및 능력치 증가
   ├─ 결과 화면 (능력치 상승량 표시)
   └─ Enter로 다음 턴 진행
   ↓
[12턴 후: 직업 선택 이벤트]
   ├─ 직업 선택 화면
   ├─ 군인/학자/연예인 중 선택
   │   └─ 잘못된 입력 시 자동으로 군인 선택 + 경고 메시지
   ├─ 취직 로딩 애니메이션
   ├─ 취직 완료 메시지
   └─ Enter로 계속 진행
   ↓
[13~24턴: 활동 선택 반복]
   ├─ 활동 선택 (직업 보너스 적용)
   ├─ 직업별 활동 메시지 출력
   ├─ 활동 실행 및 능력치 증가
   ├─ 결과 화면
   └─ Enter로 다음 턴 진행
   ↓
[엔딩 판정 (3단계 연출)]
   ├─ 1단계: 24개월 회상 (2초 대기)
   ├─ 2단계: 결과 확인 준비 (3.5초 대기)
   ├─ Enter 대기
   ├─ 3단계: 엔딩 발표 (2초 대기)
   ├─ 엔딩 이름 출력 (3.5초 대기)
   ├─ 엔딩 메시지 출력 (5초 대기)
   └─ 게임 종료 메시지

```

### 5-3. 적용된 객체지향 개념

### 1. **상속 (Inheritance)**

- `Job` 추상 클래스를 `Soldier`, `Scholar`, `Celebrity`가 상속
- 공통 기능(jobName, bonusStat)은 부모 클래스에서 관리
- 직업별 특성(보너스율, 메시지)은 자식 클래스에서 구현

```
Job (추상 클래스)
 ├── Soldier (군인)
 ├── Scholar (학자)
 └── Celebrity (연예인)

```

### 2. **추상 클래스 (Abstract Class)**

- `Job` 클래스: 모든 직업의 공통 틀 제공
- 추상 메서드: `getStatBonus()`, `getActivityMessage(String)`
- 자식 클래스가 반드시 구현해야 하는 메서드 정의

### 3. **다형성 (Polymorphism)**

- `Princess`가 `Job` 타입으로 직업을 저장
- 실행 시점에 실제 직업(Soldier/Scholar/Celebrity)에 따라 다른 동작
- 예시:
    - `princess.getJob().getActivityMessage()` → 직업마다 다른 메시지
    - `princess.getJob().getStatBonus()` → 모두 0.5 반환 (공통)

### 4. **캡슐화 (Encapsulation)**

- 모든 클래스의 필드를 `private`로 선언
- `getter`/`setter` 메서드로만 접근 가능
- Job 클래스의 필드는 `protected`로 자식이 접근 가능
- 데이터 무결성 보호

### 5. **단일 책임 원칙 (Single Responsibility Principle)**

각 클래스가 하나의 명확한 역할만 수행:

- `Princess`: 프린세스 데이터 저장 및 관리
- `Activity`: 활동 정보 저장
- `Job`, `Soldier`, `Scholar`, `Celebrity`: 직업 정보 및 동작
- `ActivityService`: 활동 처리 로직
- `GameService`: 게임 전체 흐름 제어
- `ConsoleView`: 사용자 입출력 처리

### 6. **예외 처리 (Exception Handling)**

- `InputMismatchException`: 잘못된 타입 입력 처리
- `InterruptedException`: Thread.sleep() 사용 시 예외 처리
- try-catch 블록으로 안전한 입력 처리

## 6. 구현 특징

### 6-1. 수정/추가 친화적 설계

- **단순한 클래스 구조**: 복잡한 디자인 패턴 배제
- **명확한 책임 분리**: 각 클래스의 역할이 직관적
- **주석 없이 읽기 쉬운 코드**: 메서드명과 변수명만으로 이해 가능

### 6-2. 콘솔 기반 인터페이스

- **Scanner**를 통한 사용자 입력
- **System.out.println**을 통한 출력
- **구분선 사용**: 시각적 구분을 위한 lineSplitter
- **ASCII 아트**: 타이틀 화면에 대형 로고
- **진행 애니메이션**: Thread.sleep을 활용한 시각적 효과
    - 로딩 바 (`==========>`)
    - 점 애니메이션 (`. . . . .`)
    - 취직 진행 바 (`■■■■■■`)

### 6-3. 사용자 경험 (UX) 개선

- **입력 검증**:
    - 빈 이름 입력 방지
    - 숫자 외 입력 시 재입력 요구
    - 잘못된 선택 시 명확한 오류 메시지
- **버퍼 관리**:
    - `sc.nextLine()`으로 버퍼 정리
    - nextInt() 후 남은 줄바꿈 처리
- **Enter 대기**:
    - 중요한 화면 전환 시 사용자 입력 대기
    - 스토리 진행 속도 제어

### 6-4. 확장 가능한 구조

- **새 활동 추가 용이**: Activity 배열에 추가만 하면 됨
- **새 직업 추가 용이**: Job을 상속받는 클래스 추가
- **새 엔딩 추가 용이**: showEnding() 메서드에 조건 추가
- **능력치 추가 가능**: Princess 클래스에 필드 및 메서드 추가

### 6-5. 랜덤 요소

- **Math.random()** 활용
- 주 능력치: 7~12 랜덤 증가
- 부 능력치: 2~4 랜덤 증가
- 매 플레이마다 다른 결과 제공

### 6-6. 날짜 시스템

- **LocalDate** 활용
- "yy년MM월" 형식으로 현재 날짜 표시
- 매 턴마다 자동으로 1개월 증가
- 시작: 2025년 1월 / 종료: 2026년 12월

### 6-7. 타이밍 연출

다양한 타이밍으로 게임 몰입도 향상:

- 로딩 화면: 1초 간격
- 스토리 텍스트: 2초 간격
- 활동 진행: 0.4초 간격 점 애니메이션
- 직업 선택 로딩: 0.5초 간격
- 엔딩 연출: 2~5초 다양한 대기
